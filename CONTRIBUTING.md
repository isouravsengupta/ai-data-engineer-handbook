# Contributing to AI Data Engineer Handbook

Thank you for your interest in contributing to the **AI Data Engineer Handbook** (the repository that hosts the book **AI Data Engineer Gita**). This project aims to be a high-quality, open roadmap and handbook for AI Data Engineers.

## How to Contribute

- **Typos and clarity** — Fix them via pull request.
- **New sections or chapters** — Open an issue first to discuss scope, then submit a PR against the appropriate `book/` phase.
- **Exercises and solutions** — Add under `exercises/<topic>/` with clear problem statements and (optional) solutions.
- **Projects** — Propose new project ideas in an issue; full project scaffolds go under `projects/`.
- **Research notes** — Add or extend papers in `research/` with citations and summaries.

## Pull Request Process

1. Fork the repo and create a branch from `main`.
2. Make your changes; keep formatting consistent (Markdown, line length where applicable).
3. Ensure any new code or config in `ci_cd_templates/` or `k8s_manifests/` is documented.
4. Submit a PR with a clear description; link any related issues.

## Code of Conduct

Be respectful and constructive. This is a learning resource for everyone.
