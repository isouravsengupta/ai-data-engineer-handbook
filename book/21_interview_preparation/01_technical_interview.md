# Technical Interview Guide

📄 File: `book/21_interview_preparation/01_technical_interview.md`

Guide to **technical interviews** — algorithms, SQL, ML. What to expect and how to prepare.

---

## 1 — Algorithm Interviews

* **Format**: 45–60 min, 1–2 problems
* **Focus**: Arrays, hash, two pointers, sliding window, trees, graphs
* **Process**: Clarify → approach → code → test → optimize

---

## 2 — SQL Interviews

* **Format**: 30–45 min, 2–3 queries
* **Focus**: Joins, aggregations, window functions
* **Practice**: LeetCode SQL, HackerRank

---

## 3 — ML/AI Interviews

* **Concepts**: RAG, embeddings, training vs inference
* **Design**: "Design a RAG pipeline"
* **Trade-offs**: Latency vs accuracy, batch vs streaming

---

## 4 — Common Questions

* How does a hash table work?
* Explain RAG architecture
* How would you scale a data pipeline?
* SQL: Top N per group, running sum

---

## Key Takeaways

* Clarify before coding
* Talk through your approach
* Test with examples
